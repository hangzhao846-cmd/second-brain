# 知识流转流程

目标：从输入资料走向工程资产。

```text
Inbox 捕获
→ 初步分类
→ 原子化笔记
→ 机制分析
→ 案例沉淀
→ Checklist / 模板 / ADR
→ MOC 归档
→ 周复盘 / 月复盘
→ 输出文章 / 工具 / 项目实践
```

## 1. Inbox 捕获

输入来源：技术书、官方文档、源码阅读、项目实践、线上故障、性能压测、AI 对话、文章、视频、论文、工程决策。

操作：

- 快速写下事实、上下文、链接、截图说明、问题。
- 不追求完整结构。
- 给出最小标签：`#status/todo`。

## 2. 初步分类

判断它属于哪种加工方式：

- 概念不清：概念笔记。
- 机制不明：机制笔记。
- 设计取舍：ADR 或架构笔记。
- 线上异常：故障案例。
- 指标异常：性能分析。
- 需要验证：实验记录。
- 可复用动作：Checklist。

## 3. 原子化笔记

每篇笔记只回答一个核心问题：

- 这个机制如何工作？
- 边界在哪里？
- 代价是什么？
- 失败场景是什么？
- 如何验证？

## 4. 机制分析

把资料转成自己的推理：

- 输入条件
- 内部流程
- 输出结果
- 不变量
- 边界条件
- 失败模式
- 观测指标

## 5. 案例沉淀

把实践问题沉淀成可复用案例：

- 现象
- 影响面
- 时间线
- 根因
- 证据
- 修复
- 预防
- 对现有 Checklist 的更新

## 6. Checklist / 模板 / ADR

当一类动作重复出现两次以上，就资产化：

- 排查路径写成 Checklist。
- 设计取舍写成 ADR。
- 分析套路写成模板。
- AI 协作方式写成 Prompt。

## 7. MOC 归档

把稳定笔记挂到相关 MOC：

- 机制类进入主题核心机制。
- 案例类进入案例区。
- Checklist 进入资产入口。
- 问题类进入待研究问题。

## 8. 周复盘 / 月复盘

复盘重点：

- 哪些问题被真实解决？
- 哪些知识变成工程资产？
- 哪些判断被验证或证伪？
- 哪些 AI 输出可靠或不可靠？

## 9. 输出与实践

最终输出：

- 技术文章
- 架构方案
- 排障 Runbook
- 压测方法
- 工具脚本
- Prompt 库
- 项目实践改进
