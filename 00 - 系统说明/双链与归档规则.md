# 双链与归档规则

## 什么时候使用双链

- 笔记之间存在因果、依赖、对比、案例支撑关系。
- 某个概念会被多个主题复用。
- 某个故障案例能支撑一个机制判断或 Checklist。
- ADR 依赖某个性能分析或实验结论。

示例：

- `[[机制 - JVM - Safepoint]]` 支撑 `[[性能 - 游戏服 - Stop-The-World 抖动]]`
- `[[故障 - 2026-05-31 - 网关延迟尖刺]]` 反向沉淀到 `[[Checklist - Netty 服务排障]]`

## 什么时候使用标签

- 需要跨目录检索同类笔记。
- 需要复盘某类能力增长。
- 需要查询状态，如 `#status/review`。

标签回答“它属于什么集合”，双链回答“它和谁有具体关系”。

## 什么时候进入 MOC

- 这篇笔记已经有长期价值。
- 它能支撑一个主题地图中的判断。
- 它是某类资产的入口。
- 它代表一个开放问题或关键结论。

## 什么时候创建新笔记

满足任一条件即可创建：

1. 主题会被复用。
2. 内容超过一个清晰问题。
3. 需要独立追踪验证。
4. 能沉淀为资产。

不要因为一个名词就创建空笔记。先放在当前笔记中，等它出现第二次或需要独立推理时再拆出。

## 如何避免碎片化

- 每篇笔记必须有“解决什么问题”。
- 每篇长期笔记必须连接至少一个 MOC 或项目。
- Inbox 中的原始摘录不直接进入长期目录。
- 阅读笔记只保留自己的判断、问题、实验计划和可复用结论。

## Inbox 到长期资产

1. 捕获到 `00 - Inbox`。
2. 每周清理，删除低价值内容。
3. 拆成概念、机制、案例、ADR、实验或 Checklist。
4. 挂到对应 MOC。
5. 如果能指导行动，复制或重构到 `09 - 工程资产`。
6. 在周复盘中记录“本周新增资产”。
