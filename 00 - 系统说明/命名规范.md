# 命名规范

命名目标：一眼看出笔记类型、主题和检索关键词。

| 类型        | 格式                     | 示例                            |
| --------- | ---------------------- | ----------------------------- |
| MOC       | `主题 MOC`               | `Java JVM MOC`                |
| 概念笔记      | `概念 - 名称`              | `概念 - Happens-Before`         |
| 机制笔记      | `机制 - 对象 - 机制`         | `机制 - JVM - Safepoint`        |
| 架构笔记      | `架构 - 系统 - 主题`         | `架构 - MMORPG - 网关与场景服职责边界`    |
| 故障案例      | `故障 - YYYY-MM-DD - 现象` | `故障 - 2026-05-31 - 登录服连接耗尽`   |
| 性能分析      | `性能 - 系统 - 异常指标`       | `性能 - 战斗服 - P99 延迟抖动`         |
| ADR       | `ADR-四位编号 - 决策主题`      | `ADR-0001 - 选择 Netty 作为网关网络层` |
| Prompt    | `Prompt - 任务`          | `Prompt - 代码阅读提问框架`           |
| Checklist | `Checklist - 场景`       | `Checklist - Java 服务上线前检查`    |
| 项目复盘      | `项目复盘 - 项目 - 阶段`       | `项目复盘 - 战斗服压测 - 第一阶段`         |
| 每周复盘      | `周复盘 - YYYY-Www`       | `周复盘 - 2026-W22`              |
| 每月复盘      | `月复盘 - YYYY-MM`        | `月复盘 - 2026-05`               |
| 每季度复盘     | `季度复盘 - YYYY-Qn`       | `季度复盘 - 2026-Q2`              |

## 规则

1. 标题包含可搜索关键词，不使用含糊词，如“总结”“资料”“学习笔记”。
2. 同一主题的连续研究使用同一 MOC 聚合，不靠目录层级无限拆分。
3. 日期只用于事件、复盘、实验和 ADR，不用于稳定概念。
4. 资产类笔记标题要体现使用场景。
