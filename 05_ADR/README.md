# 05_ADR

ADR 用来记录有约束、有权衡、可复盘的工程决策。

## 命名

```text
NNNN-short-decision-title.md
```

例如：

```text
0001-use-single-thread-room-model.md
0002-battle-settlement-idempotency.md
```

## 约束

- 必须写清楚背景、备选方案、最终决策和验证方式。
- 不写成事后总结，也不写成无约束的方案清单。
- 状态建议使用 `proposed`、`accepted`、`deprecated`。
